Quick Search opens instantly and stays smooth on large histories; copy text out of any image in exact layout or cleaned form; resize, convert, and remove backgrounds from image clips; full keyboard navigation in the Library; compressed hardened backups; focus returns to your app when Quick Search closes.

### Added

- **Copy text from images** — "Copy Text from Image" on image/screenshot clips with **Exact Layout** (indentation, columns, and blank lines preserved as seen in the image) and **Cleaned Up** (normalized whitespace) modes, in right-click menus and the detail panel
- **Layout-preserving OCR** — recognized text is rebuilt from Vision bounding boxes onto a character grid instead of collapsing into one line per fragment
- **Image actions** — Resize & Copy (percentages or fit-to-pixels), Remove Background (on-device subject lift), Convert & Save (PNG/JPEG/TIFF)
- **Library keyboard navigation** — arrows move selection (column-aware), Return copies, Space Quick Looks, Delete removes, Escape closes the detail panel
- **Quick Search filter cycling** — Tab / Shift+Tab cycle the type-filter chips
- **First automated test suite** — 36 standalone unit/integration tests run via `xcodebuild test`
- **Compressed backups** — `.qcclips` export (zlib), legacy JSON imports still work; version-guarded and integrity-checked

### Changed

- **Quick Search opens instantly** — the on-open list comes from an index-backed fetch touching only visible rows; no more recomputing while hidden; search-text caching no longer walks categories per keystroke; recent clips pre-warm in the background
- **Focus returns to your app** — closing Quick Search hands focus back so ⌘V works immediately in copy-only mode
- **More reliable auto-paste** — waits for the target app to actually activate (up to ~600 ms) instead of a fixed 50 ms delay
- **Faster database queries** — SQLite indexes on dedupe hash, timestamps, and content type
- **Sensitive screenshots under "Don't save"** are now removed from history after OCR (kept redacted if you favorited/pinned them meanwhile)

### Security

- Recognized image text runs through the sensitive-content scanner — screenshots of API keys/cards are caught
- Quick Look respects sensitive-clip redaction (deliberate reveal first)
- Backup import hardened against decompression bombs (size caps, streaming inflate, safe-default alerts)
